# Claude Weekly Update — April 6, 2026

> Covering the week of March 30–April 6, 2026

---

## Claude Code

### Shipped

- **Windows Desktop Control** (April 3) — Claude can now control Windows desktops (open apps, navigate browsers, fill spreadsheets) via Cowork and Claude Code on Pro and Max plans. Previously macOS only.
- **PowerShell Tool preview** (v2.1.84) — Opt-in preview for Windows developers, enabling native Windows capabilities (file system, registry, Azure/Active Directory) without routing through WSL.
- **Write tool 60% faster on large files** (v2.1.92) — Diff computation significantly faster on files containing tabs, `&`, or `$`.
- **`SessionStart` hook deferred** — Startup ~500ms faster; SessionStart hooks no longer block the initial render.
- **`@` file mentions faster** — Pre-warmed index with session-based caching and background refresh.
- **`/release-notes` interactive version picker** — Now lets you navigate to any past version's notes.
- **`/cost` per-model breakdown** — Shows cache-hit cost breakdowns for subscription users.
- **MCP tool result persistence override** — `_meta["anthropic/maxResultSizeChars"]` allows up to 500K characters per result; useful for large DB schemas.
- **Multi-line deep links** — `claude-cli://open?q=` now accepts encoded newlines.
- **Interactive Bedrock setup wizard** — Added to the login screen when selecting "3rd-party platform".
- **Remote Control session naming** — Defaults to `hostname-<random>`; overridable via `--remote-control-session-name-prefix`.
- **Plugins can ship executables** — Binaries under `bin/` are now callable as bare commands from the Bash tool.
- **Pro plan cost hint** — Footer shows estimated uncached token cost when returning to a session after cache expiry.

### Policy & Settings

- **`forceRemoteSettingsRefresh`** — New managed setting that blocks startup until remote settings are freshly fetched (fail-closed behavior).
- **`disableSkillShellExecution`** — Disables shell execution in skills, slash commands, and plugin commands.

### Key Bug Fixes

- Fixed `ctrl+e` cursor jump in multiline prompts.
- Fixed Homebrew update prompts using incorrect release channel.
- Linux sandbox: `apply-seccomp` helper now ships in both npm and native builds, restoring unix-socket blocking.
- Voice mode: fixed input lag, false "No speech detected" errors, and stale transcript re-filling on submission.

### Version Range This Week

v2.1.84 → v2.1.92

## Community Buzz & Interesting Use Cases

- **Claude Code source code leak** (March 31–April 1) — v2.1.88 of `@anthropic-ai/claude-code` was published to npm with a 59.8 MB JavaScript source map accidentally bundled in, exposing 1,906 TypeScript files and 512K+ lines of source code for ~3 hours. Anthropic confirmed no customer data or credentials were exposed. The incident lit up Hacker News with threads dissecting the architecture: message compaction flags, agent orchestration, 44 feature flags, and hidden features including KAIROS, autoDream, Undercover Mode, ULTRAPLAN, and a `/buddy` Tamagotchi companion.
- **Malware campaign riding the hype** — Security researchers found threat actors advertising fake "enterprise-unlocked" Claude Code binaries on GitHub that deployed Vidar and GhostSocks infostealers. (BleepingComputer)
- **`/buddy` Easter egg** — `/buddy` hatches an ASCII pet (18 species: dragon, duck, capybara, etc.), flagged internally as an April Fools' feature (`friend-2026-401`).
- **Usage limits still the top complaint** — r/ClaudeAI's top recurring thread: Pro plan ($20/month) runs dry after ~12 heavy sessions. Community is requesting a usage dashboard.
- **Claude Code vs. Codex** — Comparative threads consistently rank Claude Code higher on output quality; Codex wins on rate limits and cost for high-volume use.
- **Enterprise workflow adoption** — Excitement around Windows desktop control + Microsoft 365 connectors for replacing manual Office workflows.
